# Prism 深度掃描報告

## 1. 掃描摘要

Prism 目前的 current truth 是 Go primary 單一 runtime，React/Vite 前端由 Go artifact 內嵌服務，資料以 SQLite WAL + FTS5 + external data dir 儲存。整體測試基礎很厚，本次 `pytest tests/ -v`、`go test ./...`、frontend build、Loop gate 與本機 Go artifact smoke 都已跑過並通過。主要資料流、migration、FTS rebuild、path traversal、SSRF、query-only/read-write flag boundary 都有既有 regression 覆蓋，沒有發現會立即阻擋啟動或造成已知資料損毀的 P0。最大未修風險是前端 markdown rendering 直接把 `marked()` 輸出寫入 `dangerouslySetInnerHTML`，在本機同源 API / WebView 使用情境下應視為 P1。這次已完成四個低風險小修：request log 不再記 query string、CSRF origin 改成精確 origin 比對、中文/emoji 搜尋 query 裁切不破壞 UTF-8、artifact smoke 不再硬編舊 migration version。建議暫時凍結大功能，先用小步修補安全與資料一致性邊界，再補缺口測試。下一個最小行動應該是修 markdown sanitization 並鎖 ReadingView / EditablePreview XSS regression。

## 2. 專案理解

- 技術棧：Go primary REST API + SQLite/FTS5/WAL，React 18 + TypeScript + Vite + Zustand + Tailwind，Python 只保留測試/工具，不是產品 backend runtime。
- 啟動方式：本機以 `scripts/start_go_primary.ps1` 或 build artifact 啟動；Windows desktop shell 內嵌同一個 Go runtime；Pi 走 `prism-go-primary.service` + Caddy，見 `DEPLOY-PI.md`。
- 核心資料流：Browser/WebView -> React SPA -> `/api/*` -> Go handlers -> SQLite tables / FTS triggers / data-dir files。
- 主要入口：`go-shadow/main.go:156`、runtime config `go-shadow/main.go:408`、route handlers 集中於同檔；frontend API client 在 `frontend/src/services/api.ts`。
- 匯入/儲存/索引/搜尋：notes/categories/tags/attachments 由 Go API 寫 DB；notes 走 `Notes_FTS` 與關聯欄位搜尋；附件 metadata 與 bounded text attachment body scan 也會進搜尋條件。
- 更新/刪除：notes 更新走 transaction、history、tag/url replacement；刪除會清關聯表與可判定 orphan file；FTS integrity/rebuild 是手動 maintenance surface。
- 重要設定：`go-shadow/go.mod`、`frontend/package.json`、`pytest.ini`、`.loop/manifest.md`、`.loop/verify-gate.ps1`、`DEPLOY-PI.md`、`docs/CONTRACTS.md`。
- 測試方式：`pytest tests/ -v`、`cd go-shadow && go test ./...`、`cd frontend && npm run build`、`.loop/verify-gate.ps1`，以及 `scripts/smoke_go_local_artifact.ps1` 做 artifact/API smoke。

## 4. 重大問題

### P1 - Markdown rendering 未做 HTML sanitization

* 嚴重度：P1
* 位置：`frontend/src/components/ReadingView.tsx:27`、`frontend/src/components/ReadingView.tsx:426`、`frontend/src/components/editor/EditablePreview.tsx:157`、`frontend/src/components/editor/EditablePreview.tsx:139`
* 現象：ReadingView / EditablePreview 直接呼叫 `marked(markdown)`，再把結果放進 `dangerouslySetInnerHTML`；掃描未看到 DOMPurify 或等效 sanitizer。
* 原因：Markdown renderer 路徑保留 HTML rendering，但 Prism note/import/attachment 內容可由使用者或外部 Agent 匯入。
* 影響：惡意 note 內容可能在本機 browser/WebView 中執行 script 或 event handler，進一步呼叫同源 `/api/*`。Prism 沒有內建 auth，安全邊界仰賴 localhost/trusted LAN/CSRF，因此 stored XSS 的實際破壞面比一般純閱讀 preview 高。
* 建議修法：在兩個 render path 前加入本地 sanitizer，或設定 `marked` renderer 明確拒絕 raw HTML / unsafe URL；新增 regression 覆蓋 `<script>`、`onerror`、`javascript:` link、iframe/svg、中文 markdown 混排。
* 是否已修：否。本次未新增 frontend dependency 或改 markdown contract，避免在掃描任務中擴大行為面。
* 驗證方式：目前證據是 source scan；尚未跑瀏覽器 exploit regression。修補後應跑 `npm run build`、相關 frontend regression、Playwright smoke。

## 7. 建議下一步

### 立即處理

1. 修 ReadingView / EditablePreview markdown sanitization，補 XSS regression；這是目前唯一 P1。
2. 補 attachment upload hard limit，讓 upload 端與 read 端 1MiB 限制一致。
3. 檢查 DB backup 在 WAL active write 下是否能保證最新一致 snapshot；若不能，改 online backup 或 checkpoint 策略。

### 之後有空再做

1. category update/delete 的 invalid payload / target validation 改成 400/404，避免 500。
2. 大量附件搜尋時回傳 partial scan 訊號，或至少文件標註 200 files / 5MiB / 250ms boundary。
3. 針對 port occupied、Windows 中文路徑、DB lock/concurrent request、壞 DB 啟動加 smoke/regression。
4. 視實際改動需求，把 `go-shadow/main.go` 逐步拆 route group；沒有任務驅動時不要做大拆。

### 不建議現在做

1. 不建議新增登入、多租戶、雲端同步、權限系統；Prism 現在定位是本機/私有 trusted use，先把現有邊界補實。
2. 不建議導入 semantic search / AI / embedding；repo contract 明確是純關鍵字 FTS，且禁止 AI/ML 依賴。
3. 不建議為 backup 直接做大型平台化備份系統；先修 DB snapshot 語意與 data snapshot 文件/工具邊界。
4. 不建議一次性重寫 Go runtime 架構；目前測試覆蓋足夠支撐小修，但大改會提高 regression 風險。

## 8. 測試缺口

- Markdown sanitization：`<script>`、`onerror`、`javascript:`、`iframe/svg`、HTML comments、中文 markdown 混排。
- 空資料庫：fresh init 後 notes/categories/search/server status 的空狀態 API。
- 中文搜尋：中文、emoji、全形符號、混合英文大小寫、超長查詢。
- 大量文件：大量 notes / tags / attachments 下的 pagination、FTS、memory/time boundary。
- 更新/刪除後搜尋結果：note update/delete、attachment add/delete、category/tag rename 後 FTS 與關聯搜尋同步。
- 檔案不存在：attachment DB row 指向不存在檔案、upload 被外部刪除、broken image cleanup。
- Windows 路徑：空白、中文、保留字、drive letter、UNC、symlink/junction、case-insensitive collision。
- 特殊字元：filename、category/tag/note title、URL、markdown image alt text。
- API 錯誤輸入：wrong JSON type、unknown enum、negative id、limit/offset 邊界、oversized body。
- Port 被占用：desktop shell / `start_go_primary.ps1` 的錯誤訊息與 process cleanup。
- DB lock/concurrent request：同時 write、backup、FTS rebuild、VACUUM、import/export。
- Backup restore：WAL 最新交易、uploads/attachments 非 DB 檔案、restore pending marker crash recovery。

## 9. 總結判斷

這個專案可以進入暫時穩定期，但不應馬上開大功能。沒有發現已驗證的啟動阻擋、主要 API 不可用或資料損毀 P0。阻擋公開或半公開使用的是 markdown stored XSS 風險，以及 no-auth boundary 必須繼續嚴格限在 localhost/trusted LAN/VPN/proxy auth。值得繼續小修，因為現有測試/gate 能支撐低風險迭代。下一個最小行動是修 markdown sanitization 並新增對應 regression，完成後跑 `npm run build`、targeted frontend tests、`pytest tests/ -v`、`go test ./...` 與 Loop gate。
