# 4. 核心任務描述 (Core Task Description)
# -------------------------------------------------------------------
你的任務是：**依據「評估報告」的建議，對上述三份文件進行「破壞性更新 (Breaking Changes)」與修正。**

請執行以下具體操作：

1.  **修訂 `Local Insight.md` (規格書)**：
    * **API 變更**：在 API 規格中強制加入「分頁機制 (Pagination)」，定義 `page` 與 `per_page` 參數。
    * **非功能需求**：新增「離線可用性 (Offline First)」章節，明確規定前端資源 (Vue/Tailwind) 必須本地化，不可依賴 CDN。
    * **安全性**：新增「上傳安全規範」，要求檢查 Magic Numbers。

2.  **修訂 `SCHEMA.md` (資料庫)**：
    * (若報告指出資料庫結構需調整，請在此修正。若無結構性變更，請保持原樣但增加效能相關的備註，例如索引策略)。

3.  **修訂 `TODO.md` (待辦清單)**：
    * **新增修復任務**：將報告中的「優先修復」項目（如：實作 API 分頁、修正 GROUP_CONCAT 逗號 bug、下載靜態資源）插入到適當的 Phase 中。
    * **安全性任務**：新增「實作檔案上傳魔術數字檢查」、「關閉 Debug 模式」等任務。
    * **重排序**：確保高風險修復項目優先於低優先級的功能開發。
