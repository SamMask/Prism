# UX 使用者體驗分析報告：小白使用者的「輕量版」安裝之旅

**報告日期**：2025-12-13
**分析版本**：Prism v1.2 (Lightweight / Portable)
**目標受眾**：完全不懂技術、不知道什麼是 Python、終端機或環境變數的「小白使用者」
**測試目標**：驗證使用「輕量版壓縮檔」能否達成「一次完成 (One-Shot)」的安裝體驗

---

## 1. 執行摘要 (Executive Summary)

**結論：輕量版 (Lightweight) 無法對所有小白使用者保證「一次完成」。**

雖然目前的 `start.bat` 腳本已經寫得非常智慧（具備自動偵測、Winget 自動安裝、離線依賴包），但對於**沒有 Python 環境**的純淨 Windows 電腦，使用者仍需經歷「安裝 Python」的中斷過程。

**真正的「小白救星」是完整版 (Portable)。** 只有內嵌了 Python 環境的完整版，才能真正實現「解壓即用，一次成功」。

---

## 2. 使用者畫像：小白 (The Absolute Novice)

為了模擬真實體驗，我們定義一位「小白」使用者的特徵：

- **知識背景**：只會用瀏覽器、Office，以及解壓縮 zip 檔。
- **恐懼點**：看到黑色視窗 (CMD) 會緊張，看到英文錯誤訊息會直接關掉。
- **預期心理**：這就像下載一個遊戲或 Line 一樣，點兩下就能動，不要問我什麼設定。
- **環境**：一台全新的 Windows 10/11 筆電，除了 Chrome 什麼都沒裝。

---

## 3. 使用者旅程地圖 (User Journey Map) - 輕量版

我們模擬小白下載 `Prism_v1.2_Light.zip` 後的真實操作流程：

### 階段一：解壓縮與啟動

- **動作**：使用者解壓縮，看到一堆檔案。直覺雙擊 `start.bat`。
- **體驗**：✅ 檔案結構清晰，`start.bat` 命名明確。

### 階段二：環境檢測 (關鍵分歧點)

#### 🔴 情境 A：電腦沒有安裝 Python (大多數小白)

1.  **系統反應**：黑窗跳出 `[警告] 未偵測到 Python！`。
2.  **使用者心理**：「蛤？這是什麼？我壞了嗎？」
3.  **腳本救援**：`start.bat` 很聰明地跳出選項：
    ```
    請選擇安裝方式：
      [1] 自動安裝 (使用 winget...)
      [2] 手動下載
    ```
4.  **小白反應**：
    - **幸運的小白**：敢按 `1`。如果 Winget 運作正常，Python 安裝成功。**但需重新執行 `start.bat`。** -> **體驗中断**。
    - **保守的小白**：按 `2` 或直接關閉。去官網下載安裝包，**忘記勾選 "Add to PATH"** (經典陷阱)。安裝完後再執行 `start.bat` 仍然失敗。 -> **體驗崩潰**。

#### 🟢 情境 B：電腦剛好有 Python (例如曾裝過其他軟體)

1.  **系統反應**：`檢查 Python... 找到 Python: python`。
2.  **依賴安裝**：`檢查依賴...`
    - **亮點**：腳本發現 `wheels/` 資料夾，直接**離線安裝** Flask 和 Pillow。速度極快，無需等待下載。
3.  **結果**：伺服器啟動，瀏覽器自動彈出。 -> **體驗完美**。

## 4. 技術亮點與 UX 觀察

即使輕量版對小白有門檻，但在 UX 設計上仍有許多值得稱讚的細節：

1.  **離線優先的防禦網 (The Wheels Strategy)**

    - `pack.bat` 確實將 `wheels/` 打包進去了。這意味著只要有 Python，**就算沒網路也能跑**。這對於網路不穩或不想等待 Pip 下載的使用者來說，體驗極佳。

2.  **Winget 自動化嘗試**

    - 在偵測不到 Python 時提供 Winget 選項是非常先進的做法。這是在輕量化限制下，試圖挽救 UX 的最大努力。

3.  **端口衝突自動修復**
    - 如果 5000 port 被佔用，腳本會嘗試殺掉舊進程。這避免了小白遇到「Address already in use」錯誤時的不知所措。

## 6. 建議改進方案 (Recommendations)

為了達成真正的「一次完成」，建議採取以下策略：

### ✅ 策略一：主推完整版 (Portable)

在下載頁面或宣傳時，**強烈建議**小白下載 `Portable` 版本。將 `Portable` 設為預設按鈕，`Lightweight` 設為次要連結（標註「進階 / 開發者」）。

### 🛠️ 策略二：輕量版的最後一哩路 (如果一定要推輕量版)

目前的 `start.bat` 已經做到極致。唯一能改進的是在 `README.txt` 第一行用**特大號字體**寫：

> 「如果沒裝過 Python，請直接下載完整版 (Portable)，不要下載這個！」

### 🎨 策略三：視覺化引導 (未來優化)

如果資源允許，可以寫一個極小的 `.exe` (用 Go 或 Rust 寫，僅幾百 KB) 作為 Launcher。

- 它負責檢查 Python。
- 沒有就自動下載並静默安裝 (Portable Python)。
- 有就啟動。
- 這能讓輕量版也有完整版的體驗，但開發成本較高。

## 總結

**輕量版壓縮檔**是一個**技術上優雅** (體積小、依賴自包含) 但**UX 上有門檻**的解決方案。

對於「啥都不知道的小白」，**能不能一次完成？**
**答案是：不能。** (除非運氣好)

**請直接把 `Prism_v*_Portable_*.zip` 塞給他，那才是真正的 UX 滿分。**
