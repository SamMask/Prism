# Prism Desktop Portable

This folder is the Windows desktop portable package.

## Run

Double-click `Prism.exe`.

`Prism.exe` is built as a Windows GUI executable and starts the Prism desktop shell directly: the Go runtime runs in the same process and the UI opens in WebView2 at a local `127.0.0.1` address. `Prism.ico` is generated by the build scripts, embedded into the Windows executable resource, and loaded by the desktop shell for the window and tray icon.

For diagnostics, run `PrismDesktop-debug.exe` from PowerShell. It keeps a console window and accepts the same flags:

```powershell
.\PrismDesktop-debug.exe --data-dir "D:\PrismData" --addr 127.0.0.1:5015
```

## Data

By default, the portable build stores user data next to the executable:

```text
<Prism.exe folder>\PrismData
```

Advanced/debug launches can override that location with `--data-dir` or `PRISM_GO_DATA_DIR`. The double-click portable path does not show a first-run data folder picker, does not write a portable config file, and does not create or repair desktop shortcuts.

This simplified portable baseline was manually accepted on 2026-06-18 after copying the package to another folder and running `Prism.exe`; Windows Defender did not block the executable.

During normal startup, the desktop window shows a small Prism startup screen before navigating to the local runtime URL, so users do not see an unexplained blank white window while WebView2 loads.

The data directory contains the desktop database (`prism_desktop_dev.db`), uploads, attachments, config, backups, and logs. Desktop shell logs are written to:

```text
<data-dir>\logs\desktop-shell.log
```

The portable package includes `static\config\prompt_options.json` and `static\config\wizard_options.json`. On a fresh data directory, the Go runtime seeds Prompt Builder config from those files.

## Requirement

Microsoft Edge WebView2 Runtime must be installed on the machine. If WebView2 is missing, the desktop shell exits with a diagnostic error in the debug console or desktop log. This portable package does not install WebView2 Runtime and is not an MSI/NSIS/WiX installer.

## Deferred Installer Work

These options are intentionally deferred until there is a real installer/package gate:

- choosing between portable data, a Windows account data folder, or a custom folder
- creating Start Menu or desktop shortcuts
- installing WebView2 Runtime
- uninstall/update flow

## Boundaries

- No installer or auto updater is included.
- No local production database is bundled.
- Raspberry Pi deployment is separate and still uses the linux/arm64 Go primary artifact, systemd, Caddy, and `DEPLOY-PI.md`.
