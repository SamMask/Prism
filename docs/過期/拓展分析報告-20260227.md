# Prism V2 拓展功能提案 (輕巧實用版)

## 1. 現狀診斷 (Current State Analysis)

經過對 Prism 專案架構（Flask + React + SQLite + 本地 NLP/Vision 模型）及現有功能的詳細盤點，我發現 Prism 其實已經具備了相當完整的基礎建設：
* **核心功能**：瀑布流圖庫、筆記 CRUD、Markdown 編輯器、支援拖曳上傳與貼上。
* **AI 賦能 (已實作)**：Ollama 影像自動標籤、CLIP 語意搜尋 (Semantic Search)、RAG 文件初步切塊、Llama 文字摘要。
* **特殊工具**：可視化的 Prompt Builder。

> **核心矛盾點**：您已經把「骨架」跟「大腦」都建好了（有高階的 Embedding 特徵與本地 AI），但目前的 UI/UX 依然停留在「傳統文件管理器」的模式。使用者必須「主動點擊、主動搜尋」才能獲得回饋，這樣太浪費您建置的這套 AI 基礎設施了。

因此，這次的新功能提案完全捨棄「增加新資料表」或「導入新模型」的構想，**100% 聚焦在如何用最少的 React/Flask 程式碼，把現有資料庫變成能主動啟發您的「靈感餵食器」**。

## 2. 三項高 C/P 值新功能提案

### 提案一：無限靈感側滑欄 (Inspiration Sidebar)
*在你檢視或編輯筆記時，無縫推播「似曾相識」的舊資源。*

* **功能情境**：
  當您點開一張參考圖或一則筆記，右側會自動滑出一個窄窄的面板，裡面顯示 **Top 5 相似的圖片/筆記** 或是 **包含相同 Prompt 參數的卡片**。
* **為什麼好做又實用？**
  * **技術成本極低**：後端已經有 `routes/ai.py` 的 Semantic Search 和 FTS5 了，**完全不需要寫新的後端邏輯**。
  * **前端實作**：在 `NoteEditor.tsx` 或 `NoteCard.tsx` 旁邊加一個 Drawer 元件，當 `selectedNote` 改變時，自動發起一個背景 API 請求拿相似資料展示即可。
* **價值**：徹底免除「看了一張圖覺得不錯 -> 關閉 -> 在上方搜尋列打字找類似的」這套繁瑣的流程，實現無限點擊漫遊。

### 提案二：Prompt 參數熱度分析與復用 (Prompt Heatmap & Auto-Injector)
*找出你最愛用的、最容易出好圖的 Prompt，讓你隨手可用。*

* **功能情境**：
  在您的 `PromptBuilder` 頁面新增一個「熱門統計」區塊。系統自動分析所有筆記的 `prompt_params` (JSON)，用大小不同的文字（標籤雲）顯示您這一個月內最常使用的 Prompt 關鍵字。點擊任何一個字，可以一鍵插入到當前的 Prompt 編輯區。
* **為什麼好做又實用？**
  * **技術成本低**：只需寫一支簡單的 SQL (利用 SQLite JSON 函數解析 `Notes.prompt_params`) 聚合統計數量。
  * **前端實作**：一個簡單的 Tag 陣列渲染。
* **價值**：解決「我上禮拜畫了一張光影很棒的圖，但忘了是加了哪句 Prompt」的痛點，讓過往成功經驗具象化。

### 提案三：隨機回顧按鈕 (The "Surprise Me" Button)
*最暴力的對抗遺忘曲線機制，專治「數位囤積症」。*

* **功能情境**：
  在 Header (導覽列) 加一顆簡單的骰子 🎲 按鈕。每次點擊，就從分類為「靈感」或「提示詞」中，隨機彈出一張超過一個月沒修改過的筆記/圖片 Modal。
* **為什麼好做又實用？**
  * **技術成本幾乎為零**：SQL 就是 `SELECT * FROM Notes ORDER BY RANDOM() LIMIT 1`。
  * **前端實作**：綁定一個點擊事件去觸發原本就寫好的 `NoteDetailModal`。
* **價值**：很多時候我們打開庫存只是「想隨便看看找靈感」，而不想設定搜尋條件。這顆按鈕能用最低廉的成本打破資料庫的封閉感。

## 4. 惡魔辯護人（功能反思）

> 身為提案者，我也必須指出這些小功能的侷限性，避免期望落差：

* **關於側滑欄的擁擠感**：Prism V2 是基於瀑布流架構，畫面資訊量本就不小。如果 Editor modal 又加上右側欄，在小螢幕（如 13 吋筆電）上會顯積極其擁擠。**解法**：建議預設收起，做一個半透明的箭頭讓使用者 hover 時才滑出，保持主視野清爽。
* **隨機回顧可能很無聊**：如果您的圖庫裡有很多未分類的垃圾截圖，按兩次骰子都跑出無意義的圖片，使用者就不會再按了。**解法**：隨機撈取的 Query 建議加上過濾條件，例如 `cover_image IS NOT NULL` 或被釘選 (`is_pinned`) 的優先。
