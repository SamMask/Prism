## 一、互動邊界與狀態回饋 (State & Feedback)

### 1.1 全域錯誤攔截器缺失

| | |
|---|---|
| **現狀** | `api.ts` 的 axios instance 未設定全域 `interceptors.response`。每個呼叫端各自 try/catch，導致錯誤訊息格式不統一。部分 catch block 僅 `console.error` 而無 UI 回饋 (e.g. `appStore.ts:120`, `fetchCategories`, `fetchTags`)。 |
| **目標檔案** | [api.ts](file:///d:/AI/Prism/frontend/src/services/api.ts)、[appStore.ts](file:///d:/AI/Prism/frontend/src/stores/appStore.ts) |
| **重構建議** | 在 axios instance 上加入 `client.interceptors.response.use(res => res, error => { ... })` 統一錯誤處理。區分 `401/403` (權限)、`404` (資源不存在)、`5xx` (伺服器異常)，搭配 Toast 彈出分類訊息。各元件層僅需處理「業務邏輯級」錯誤。 |
| **優先級** | **High** — 影響所有 API 互動的一致性體驗 |

## 四、無障礙與適配性 (a11y & RWD)

### 4.1 ARIA 標籤完全缺失

| | |
|---|---|
| **現狀** | **整個 V2 前端無任何 `aria-*` 屬性、無 `role` 屬性、無 `tabIndex` 設定。** 這是最嚴重的無障礙問題。<br>具體影響：<br>• Modal (`Modal.tsx`) — 無 `role="dialog"`、無 `aria-modal="true"`、無 `aria-labelledby`。<br>• 側邊欄 (`Sidebar.tsx`) — `<aside>` 標籤語意正確但無 `aria-label`。<br>• 搜尋結果下拉 — 無 `role="listbox"` / `role="option"`。<br>• NoteCard 右鍵選單 — 無 `role="menu"` / `role="menuitem"`。<br>• 所有 icon-only 按鈕 — 僅有 `title` (不等同 `aria-label`)。<br>• Toast 通知 — 無 `role="alert"` / `aria-live="polite"`。 |
| **目標檔案** | 全域性問題，特別是 [Modal.tsx](file:///d:/AI/Prism/frontend/src/components/ui/Modal.tsx)、[Toast.tsx](file:///d:/AI/Prism/frontend/src/components/ui/Toast.tsx)、[NoteCard.tsx](file:///d:/AI/Prism/frontend/src/components/NoteCard.tsx)、[Header.tsx](file:///d:/AI/Prism/frontend/src/components/Header.tsx)、[Sidebar.tsx](file:///d:/AI/Prism/frontend/src/components/Sidebar.tsx) |
| **重構建議** | 優先修復以下項目（按影響排序）：<br>**P0:**<br>1. `Modal.tsx` — 加入 `role="dialog"` + `aria-modal="true"` + `aria-labelledby={titleId}`。<br>2. `Toast.tsx` — container 加 `aria-live="polite"` + 每個 toast 加 `role="status"`。<br>3. 所有 icon-only 按鈕加 `aria-label` (e.g. 搜尋清除 `aria-label="清除搜尋"`)。<br>**P1:**<br>4. NoteCard dropdown menu — `role="menu"` + `role="menuitem"` + 鍵盤導覽 (`ArrowUp/Down`)。<br>5. `<aside>` 加 `aria-label="主導覽列"`。<br>6. 搜尋結果下拉 — `role="listbox"` + `aria-activedescendant`。 |
| **優先級** | **High** — 不符合 WCAG 2.1 AA 基本要求 |

### 4.2 鍵盤導覽不可用

| | |
