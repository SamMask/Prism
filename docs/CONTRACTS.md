# Prism Contracts

本檔是 active TODO 的契約索引。每個 active construction task 都必須指向本檔中的一個契約名稱；候選 backlog 可先不施工，promote 前必須補 contract。契約名稱只描述必須守住的 runtime 邊界，不代表該功能已完成。

| Contract | Boundary |
|---|---|
| CONTRACT-GO-PRIMARY-TODO-GOVERNANCE | `docs/TODO.md` 只保留 active roadmap、候選 backlog、下一步入口與歸檔索引；`HANDOFF.md` 只保留最短 current state / next entry；歷史 phase、完成紀錄與長版交接快照移入 `docs/development-history/`。 |
| CONTRACT-DESKTOP-SHELL-SPIKE | Desktop Shell Phase 0 只能建立 isolated Windows message-loop spike：空 Win32 視窗 + tray icon + 單一 loop；不得接 WebView2、後端、schema/API/runtime、deploy 或 production data。 |
| CONTRACT-DESKTOP-SHELL-WEBVIEW2-SPIKE | Desktop Shell Phase 1 只能在 Phase 0 message loop proof 上接入 WebView2 placeholder / URL target；不得接 Prism Go server goroutine、schema/API/runtime、deploy、installer、updater 或 production data。 |
| CONTRACT-DESKTOP-SHELL-RUNTIME-HOST | Desktop Shell Phase 2 只能把 Windows shell 接到同一行程內的 Go primary runtime goroutine，使用明確 local addr 與 external data-dir；不得碰 production Pi data、不得改 schema/API、不得改 Pi deploy、不得做 installer/updater。 |
| CONTRACT-DESKTOP-SHELL-UX-HARDENING | Desktop Shell Phase 3 只能補 Windows GUI/UX hardening：debug console build、`-H=windowsgui` GUI build、檔案 log、named mutex single-instance、tray Show/Quit 與 close exits process；不得做 portable package、installer、auto updater 或 Pi deploy。 |
| CONTRACT-DESKTOP-SHELL-PORTABLE-PACKAGE | Desktop Shell Phase 4 只能產出免安裝 Windows portable zip / folder：`Prism.exe` GUI artifact、debug artifact、portable README、exe 同層 `PrismData\` default data-dir、clean-unzip smoke 與 no-bundled-DB guard；不得做 first-run data-dir selector、portable choice config、desktop shortcut automation、installer、auto updater、Pi deploy 或 production data bundling。 |
| CONTRACT-DESKTOP-SHELL-INSTALLER-DECISION | Desktop Shell Phase 5 只做 installer/updater decision gate：portable zip 維持 Windows release 主路徑，除非 Start Menu、file association、WebView2 bootstrap、uninstall 或 update 需求明確，否則不得引入 NSIS/WiX/MSIX 或 auto updater stack。 |
| CONTRACT-DESKTOP-SHELL-PI-BOUNDARY | Desktop Shell Phase 6 鎖定 Windows desktop 與 Pi headless deployment 分離：Pi 仍使用 linux/arm64 Go primary artifact、`prism-go-primary.service`、Caddy 與 `DEPLOY-PI.md`；不得把 WebView2、Win32 tray/mutex、`-H=windowsgui`、installer 或 updater 帶入 Pi path。 |
| CONTRACT-SEARCH-UX-EXPLAINABILITY | Search UX 只能改善現有 `GET /api/notes?q=...` 的可解釋性：Search Context Bar、scope hint、empty-state recovery、mobile search entry 與 localStorage recent searches（最多 5 筆）。不得新增 relevance ranking、semantic search、attachment body snippets、SearchHistory DB、advanced query language 或 per-keystroke live search。 |
| CONTRACT-SEARCH-INTEGRITY-DIAGNOSTICS | Search integrity 只可提供手動診斷與明確按鈕觸發的 FTS rebuild：`GET /api/system/search-integrity` read-only；`POST /api/system/search-integrity/rebuild-fts` 只重建 `Notes_FTS`。不得 VACUUM、改 Notes、改 Attachments、刪檔、自動修復或讓 Agent 自動觸發。 |
| CONTRACT-MAINTENANCE-HEALTH-OVERVIEW | Maintenance health overview 只呈現現有維護狀態與手動維護入口，不新增自動修復行為、不改 DB schema/API exposure boundary、不放寬 `/api/server/*` local-only 限制。 |
| CONTRACT-VARIANT-PANEL | Variant tracking panel 只可沿用既有 `Notes.parent_id` lineage，補 `GET /api/notes?parent_id=<id>` read-only 直接 child lookup、`variants_count` 顯示欄位、ReadingView parent/child navigation 與 NoteCard affordance；duplicate-as-variant 必須保留既有 note 複製語義，包含 tags、source URLs、文字附件與長內容自動分離附件，且子 note 必須拿到自己的附件 row / file path；不得新增 DB schema/migration、版本樹、diff/merge engine、協作語義、server-side UI state 或新的 note write semantics。 |
| CONTRACT-NOTE-LIST-LIGHTWEIGHT | Home/list 只能拿卡片渲染所需的輕量 note payload：`GET /api/notes` 回 `content_preview`、`content_truncated`、`content_length`，且相容欄位 `content` 只等於 preview；`GET /api/notes/<id>` 維持完整 note detail。搜尋仍以 DB/attachment body contract 執行；前端在 Editor、copy、image export 或 ReadingView 需要完整內容時必須 lazy-load detail / auto-extracted attachment。不得新增 schema migration、cache layer、server-side UI state、全文預載、背景同步或改附件儲存根目錄。 |
| CONTRACT-STARRED-TAG-FILTERS | Header starred tag shortcuts 只可使用純前端 `localStorage` state contract（key：`prism.starredTags.v1`）保存 starred tag IDs；刪除 / merge tag 後若 ID 不存在，讀取時自動忽略。Settings tag star control 必須 stop propagation，不影響既有 merge selection、rename、delete；FilterStrip 只能顯示 starred tags，無 starred tags 時只顯示低存在感提示文字。不新增 DB 欄位、不改 tags API、不做跨裝置同步、server-side preference、tag 排序拖曳、tag 群組或 sidebar tag list redesign。 |
| CONTRACT-IMAGE-VIEWER-LIGHTBOX | Unified image lightbox 只可新增純前端 shared component 與呼叫端傳入圖片清單；ReadingView 可收集 cover image 與 rendered markdown images，Editor preview / editor gallery 可用同一 lightbox，NoteCard cover 只能透過明確 icon/button 開啟，不得 hijack 整張卡片點擊習慣。支援 click、Esc、左右鍵、copy path、open original；不得改圖片儲存模式、upload/delete/cleanup API、DB schema、gallery DB、OCR/AI 圖片描述或重寫 markdown renderer。 |
| CONTRACT-IMAGE-VIEWER-ZOOM | Lightbox zoom controls 只可在既有 shared `ImageLightbox` 內新增純觀看縮放、背景點擊關閉與 lightbox-active keyboard shortcuts；切換圖片或重開 lightbox 必須回到 100%/fit 狀態。不得持久化 per-image zoom、不得新增 crop/rotate/edit、不得改圖片 storage/upload/delete/cleanup API、附件 contract、DB schema、gallery DB、server-side preference 或 AI/OCR 圖片分析。 |
| CONTRACT-BULK-MARKDOWN-TXT-IMPORT | Batch Markdown/txt import 只可在 Settings 前端加逐檔 wrapper：`.md` 呼叫既有單檔 `POST /api/notes/import/md`，`.txt` 只可由前端讀檔後走既有 `POST /api/notes`，title 使用檔名 stem、content 使用純文字內容；前端可維護本機待匯入清單，允許多次選擇不同資料夾的檔案、重複檔案去重、單檔移除與清空，匯入後清空待匯入清單但保留結果摘要；每檔獨立成功/略過/失敗回報，單檔失敗不得回滾已建立 notes。不新增 server-side batch API、DB schema/migration、目錄 watcher、AI 摘要、自動分類、overwrite/sync/background daemon 或批量 DB transaction。 |
| CONTRACT-READING-WORKSPACE | Reading list workspace 只可使用純前端 `localStorage` state contract（key：`prism.readingWorkspace.v1`）保存 note ids、active id、layout preference（tabs/sidebar）與每張 note 的 scroll position；ReadingView 可顯示暫存清單、加入目前 note、切換、移除單張、清空全部與失敗可移除狀態；NoteCard action menu 可加入 note id，但不得強迫改變既有卡片開啟模式。不得新增 DB schema、server-side preference、跨裝置同步、native 多視窗、雙欄比對、diff engine、改 `GET /api/notes` contract 或改 note API。 |
| CONTRACT-CATEGORY-IDENTITY | 五個系統分類的身份必須由 `Categories.system_key` 表示，使用者改名只寫入 `Categories.name_override`；`Categories.name` 保留 legacy canonical 相容用途，`is_default` 只作刪除分類搬移目標。migration 只可回填仍保留完整 legacy seed name 的系統分類，不得把已自訂名稱的分類誤判回系統分類；前端顯示不得再把語系文字當 identity。 |
| CONTRACT-DEEP-SCAN-RISK-CLOSURE | Deep scan risk closure 只可用小步修補已驗證風險與測試缺口：markdown render path 必須 sanitizer-gated；文字附件 upload/read limit 必須一致；server DB backup 必須是 WAL-current consistent DB-only snapshot；category invalid input 必須回 400/404；bounded attachment body search 超限必須可診斷；no-auth/local-only exposure boundary 必須以 runtime/docs/tests 鎖住；stability packs 只補明確 fixture，不得新增 auth、AI、schema、cloud sync、大型備份平台或一次性 Go runtime 拆分。 |
| CONTRACT-GO-PRIMARY-ARCHITECTURE | 文件必須以 Go primary 為唯一 current runtime owner；歷史 candidate、sidecar、rollback proof 不得寫成 live owner，retained-Python 主路徑描述不得出現在 current docs。 |
| CONTRACT-GO-PRIMARY-ROUTE-OWNERSHIP | 所有正式 API route 必須有唯一 production owner；Go 替代 Python 前，manifest 中不得留有 Python-owned request-time surface。 |
| CONTRACT-GO-PRIMARY-PARITY | Go route promoted 前，必須有 Python-vs-Go parity fixture 證明 status、JSON shape、DB state、file state 與 rollback 行為一致。 |
| CONTRACT-GO-PRIMARY-CONFIG | Go runtime 必須使用明確 external data dir，所有 DB/uploads/attachments/logs/backups path 都不能逃逸資料根目錄。 |
| CONTRACT-GO-PRIMARY-DB | Go 必須正確擁有 SQLite connection、WAL、transaction、busy timeout、write mode 與 rollback semantics。 |
| CONTRACT-GO-PRIMARY-MIGRATIONS | Go 必須能 fresh init、existing DB upgrade、idempotent skip、failed migration rollback、backup-before-migrate，才可取代 Python migration runner。 |
| CONTRACT-GO-PRIMARY-NOTES | Notes read/write/actions/batch/history 必須完整 Go-owned，包含 FTS、關聯欄位、history、delete side effects 與 image cleanup 邊界。 |
| CONTRACT-GO-PRIMARY-TAXONOMY | Categories/tags CUD/merge/default/sort 必須完整 Go-owned，且與 schema uniqueness、NOCASE、Note_Tags 關聯行為一致。 |
| CONTRACT-GO-PRIMARY-FILES | Attachments metadata、raw/text/binary serving 與 path/MIME 安全必須由 Go 完整負責。 |
| CONTRACT-GO-PRIMARY-UPLOADS | upload、upload-url、upload delete、thumbnail generation 必須 Go-owned，並覆蓋 SSRF、MIME/magic、size、filename、rollback/no orphan file。 |
| CONTRACT-GO-PRIMARY-MEDIA-CLEANUP | orphan images、originals、broken images、note delete companion cleanup 必須 Go-owned，且不得誤刪仍被引用檔案。 |
| CONTRACT-GO-PRIMARY-IMPORT-EXPORT | Markdown/JSON/DB/images import/export 必須 Go-owned，且輸出可還原、失敗不留下 partial DB/files。 |
| CONTRACT-GO-PRIMARY-SERVER-SYSTEM | server version/status/hardware/logs/backup/port/config/service availability 必須 Go-owned，正式 runtime 不依賴 Python service。 |
| CONTRACT-GO-PRIMARY-STATIC-SERVING | SPA、static assets、uploads serving 與 API fallback 必須有 Go/Caddy 明確分工，不能因 fallback 讓 API 404/405 變成 SPA。 |
| CONTRACT-GO-PRIMARY-SECURITY | Go primary 必須保留現有 exposure boundary，並鎖住 path traversal、SSRF、MIME、size、public internet warning 與 no-auth 風險。 |
| CONTRACT-GO-PRIMARY-PACKAGING | Windows 與 linux/arm64 package smoke 必須證明產品啟動不需要 Python/venv/Flask/PyInstaller runtime。 |
| CONTRACT-GO-PRIMARY-DEPLOY-CUTOVER | live cutover 必須包含 backup、systemd primary switch、Caddy route switch、frontend default API 驗證與 full workflow smoke。 |
| CONTRACT-GO-PRIMARY-ROLLBACK | rollback 必須能從 Go primary 還原 Python runtime、Caddy、systemd、DB/files，並有 live smoke 證據。 |
| CONTRACT-GO-PRIMARY-PYTHON-DELETION | Go primary cutover、rollback、soak、package smoke 全部通過後，才允許刪除 Python packaged runtime；Python backend source 的最終封存/刪除必須另由 T053 完成。 |
