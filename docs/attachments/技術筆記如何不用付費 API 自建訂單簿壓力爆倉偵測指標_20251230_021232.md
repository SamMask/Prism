# **技術筆記：如何不用付費 API 自建「訂單簿壓力／爆倉偵測指標」**

### 🎯 目標：

建立一組能反映市場潛在清算風險與壓力集中區的微觀指標，不依賴 Coinglass 等付費資料源。

---

## ✅ 一、訂單簿壓力指標（Orderbook Pressure / Imbalance）

### 🛠 所需來源：

- **Binance API**
    
    - `GET /api/v3/depth?symbol=BTCUSDT&limit=1000`
        
    - 或用 WebSocket：`wss://stream.binance.com:9443/ws/btcusdt@depth20`
        

---

### 📐 指標 1：±x% 區間買賣量差值

#### 步驟：

1. **取得中間價：**
    

python

複製編輯

`mid_price = (best_bid + best_ask) / 2`

2. **定義±x%的價格區間（例如 ±0.5%、±1%、±2%）：**
    

python

複製編輯

`lower_bound = mid_price * (1 - 0.005) upper_bound = mid_price * (1 + 0.005)`

3. **統計區間內的買賣盤總量：**
    

python

複製編輯

`buy_qty = sum(qty for price, qty in bids if lower_bound <= price <= mid_price) sell_qty = sum(qty for price, qty in asks if mid_price <= price <= upper_bound)`

4. **計算壓力指標：**
    

python

複製編輯

`orderbook_pressure = (buy_qty - sell_qty) / (buy_qty + sell_qty)`

#### ➕ 優點：

- 可反映當下主動掛單偏多還是偏空
    
- 實時更新（搭配 WebSocket）
    

#### 🧠 延伸應用：

- 壓力反轉：若 `±1%` 壓力方向突變，可作為爆倉或止損群體觸發的信號
    

---

## ✅ 二、主動成交方向指標（Aggressive Flow）

### 🛠 所需來源：

- `GET /api/v3/aggTrades?symbol=BTCUSDT&limit=1000`
    

#### 🧮 處理步驟：

1. 按時間排序
    
2. 對每筆 `aggTrade` 計算：
    

python

複製編輯

`buy_volume = sum(qty for trade in trades if trade['m'] == False) sell_volume = sum(qty for trade in trades if trade['m'] == True) aggressive_flow_ratio = (buy_volume - sell_volume) / (buy_volume + sell_volume)`

- `m == True` 代表 **賣方主動成交**（打到買單）
    
- `m == False` 是 **買方主動成交**
    

#### 🔎 解讀：

- 正值代表市場主動買入強，負值代表主動賣出壓力大
    
- 突然從正轉負 → 可能是清算瀑布或市場恐慌
    
- 可搭配 K 線轉折判斷是否「主動砸盤」

## ✅ 三、合約爆倉熱區（自建）

雖然不能取得清算實際發生點，但可以模擬「爆倉可能區域」的方法如下：

### 方法一：**用槓桿多空倉位 + 開倉價計算爆倉線**

若有多空比資料，可估計可能爆倉區間（但目前沒付費，建議跳過）

---

### 方法二：**從「買賣壓 + 價格突刺 + 主動賣出暴增」三重條件偵測爆倉觸發**

#### 條件偵測：

python

複製編輯

`if orderbook_pressure < -0.8 and aggressive_flow_ratio < -0.7 and price_change < -1.5%:     trigger = '可能出現爆倉連鎖反應'`

---

## ✅ 四、爆倉影子指標（Post-Liq Proxy）

1. 計算每分鐘：
    
    - 成交量異常增加
        
    - 主動賣出占比超過 80%
        
    - 買賣壓力指標突然急轉
        
2. 過去 5 分鐘內若出現此種結構，可標記為「爆倉陰影區」
