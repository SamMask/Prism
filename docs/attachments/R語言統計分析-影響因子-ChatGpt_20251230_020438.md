## **② 影響度分析（選擇有意義的因子）**

你的目標是找出「哪些因子對最終收益或價格走勢最有影響」。  
可以分成幾類方法：

### 1. **相關性檢定**

- `cor()` / `Hmisc::rcorr()` → 計算皮爾森或斯皮爾曼相關係數
    
- 缺點：只能抓線性關係，且不能反映時間延遲
    

### 2. **單變量檢定**

- `tseries` / `urca` → 單根檢定（避免假相關）
    
- `lm()` → 單因子回歸，檢定顯著性（p-value）
    

### 3. **多因子回歸 / 特徵選擇**

- `glmnet`（Lasso / Ridge 回歸）：自動做變數篩選與正則化，適合多變量+防止過擬合
    
- `randomForest`（隨機森林）：可輸出變數重要度 ranking
    
- `xgboost` / `lightgbm`（梯度提升樹）：擅長非線性關係，輸出 feature importance
    
- `Boruta`（穩健特徵選擇法）

## **③ 權重優化與交易決策**

確定哪些因子有用後，要計算權重 → 下單數量。

### 1. **權重計算方法**

- **回歸係數權重法** → 使用 `glmnet`、`lm()` 的係數比例作為權重
    
- **均方誤差最小化** → `optim()`、`nloptr` 做權重優化
    
- **均值-方差模型** → `PortfolioAnalytics` / `quadprog`（像做投資組合優化一樣）
    
- **風險平價法** → 權重與風險貢獻相等（適合波動大的因子）
    

### 2. **下單決策模型**

- **信號加權打分法**（score > 閾值 → 開單）  
    範例：
    
    scoret=∑i=1nwi⋅signaliscore_t = \sum_{i=1}^n w_i \cdot signal_iscoret​=i=1∑n​wi​⋅signali​
    - signal 例：RSI < 30 → +1，多空比高 → -1
        
    - 權重 wiw_iwi​ 來自上一步
        
- **機器學習預測法** → 預測未來 Δ價格，再決定方向與下單量  
    套件：`caret`、`tidymodels`、`xgboost`
    

### 3. **倉位管理**

- `RQuantLib` → 根據風險值（VaR、CVaR）決定下單量
    
- 動態槓桿法：槓桿 ∝ 預測信號強度 × 風險承受度

library(tidyverse)
library(quantmod)
library(glmnet)
library(randomForest)

# 1. 整合數據
data <- merge(price_data, rsi_data, orderbook_spread, liq_data, 
              long_short_ratio, open_interest, app_rank, by="timestamp")

# 2. 特徵選擇（Lasso）
x <- as.matrix(data %>% select(-target_return))
y <- data$target_return
lasso_model <- cv.glmnet(x, y, alpha=1)
coef(lasso_model)

# 3. 權重打分
weights <- coef(lasso_model)[-1] # 去掉截距
data$score <- x %*% weights

# 4. 交易決策
data$position <- ifelse(data$score > 0.5, 1,
                        ifelse(data$score < -0.5, -1, 0))

# 5. 績效回測
library(PerformanceAnalytics)
returns <- data$position * data$target_return
charts.PerformanceSummary(xts(returns, order.by=data$timestamp))

## 關鍵操作細節

### 1) 資料對齊

- **頻率一致**：建議以決策週期為準（例：1 分鐘或 15 分鐘）；日線 RSI 可轉為低頻特徵（上次日線收盤值）。
    
- **缺值處理**：前值填補（carry-forward）或以統計值補齊；記錄缺值比例避免偏誤。
    
- **標準化**：`scale()` 或以 robust scaler（中位數/四分位）處理厚尾資料。
    

### 2) 特徵工程

- **滯後與變化率**：lag_1、lag_5、Δ1、Δ5、rolling z-score。
    
- **去趨勢**：對長期趨勢明顯的指標（如 App 排名、COT）可做差分或 HP 濾波（需謹慎）。
    
- **泄漏檢查**：嚴禁使用未來資料（look-ahead）。
    

### 3) 因子挑選與檢定

- **穩健性**：在滾動視窗裡重複驗證重要度是否穩定。
    
- **多重共線性**：VIF 檢查，或以 `glmnet` 正則化緩解。
    
- **非線性/交互**：樹模型（RF/XGB）發掘；對重要交互再回到線性框架做可解釋化。
    

### 4) 權重與倉位

- **三條路**：
    
    1. 以 **Lasso 係數** 直接定權重（快、可解釋）。
        
    2. 以 **均值–方差/風險平價** 做權重（穩定、對抗波動）。
        
    3. 以 **自訂目標函數**（最大化夏普、限制回撤、交易成本懲罰）用 `nloptr/optim` 搜尋。
        
- **倉位連續化**：`position = clamp(k * score / vol_target, -maxLev, maxLev)`。
    
- **交易成本**：滑點 & 手續費必須內生於回測評估。
    

### 5) 回測與評估

- **滾動回測**：Walk-Forward（訓練→驗證→前推）。
    
- **評分面板**：年化報酬、夏普、Calmar、最大回撤、勝率、偏度、尾風險（CVaR）。
    
- **穩健檢查**：跨區間、不同頻率、子市場（週末/工作日、亞洲/美歐時段）。

# 0) 準備 ----
library(data.table); library(tidyverse); library(lubridate)
library(xts); library(TTR); library(glmnet); library(PerformanceAnalytics)

# 1) 整合資料（需先對齊時間戳）
DT <- reduce(list(price, rsi_d, ob_1pct, liq, lsr, oi, cme_cot, app_rank),
             ~ merge(.x, .y, by = "timestamp", all = TRUE)) |> as.data.table()
DT <- DT[order(timestamp)][, lapply(.SD, zoo::na.locf, na.rm = FALSE),
                           .SDcols = !c("timestamp")]

# 2) 目標定義（未來報酬）
h <- 5 # 例如預測未來 5 根 bar 報酬
DT[, ret := shift(log(price) - log(shift(price, n = 1)), type = "lead", n = h)]

# 3) 特徵工程
feat_cols <- setdiff(names(DT), c("timestamp", "ret"))
X <- scale(as.matrix(DT[, ..feat_cols]))
y <- DT$ret

# 4) 滾動訓練
win <- 2000; step <- 100
pos <- rep(NA_real_, nrow(DT))
for (i in seq(win, nrow(DT) - step, by = step)) {
  tr_idx <- seq(i - win + 1, i)
  te_idx <- seq(i + 1, i + step)
  fit <- cv.glmnet(X[tr_idx,], y[tr_idx], alpha = 1)
  w <- as.numeric(coef(fit))[-1]
  s <- X[te_idx,] %*% w
  # 波動標準化
  vol <- sd(y[tr_idx]); k <- 1/vol
  pos[te_idx] <- pmax(pmin(k * s, 1.5), -1.5)
}

# 5) 回測
ret_xts <- xts(pos * y, order.by = as.POSIXct(DT$timestamp))
charts.PerformanceSummary(ret_xts)
